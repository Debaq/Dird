# Preferencias del Proyecto

## Documentación
- **NO crear archivos .md** a menos que se solicite explícitamente
- El usuario prefiere comunicación directa sin documentación innecesaria

## Estilo de Código
- YOLOv11n para detección (no YOLOv8)
- TypeScript + React + Vite
- Zustand para estado global
- IndexedDB (Dexie.js) para persistencia

## Flujo de Trabajo
- Implementar funcionalidad primero
- Documentar solo cuando se pida
- Probar con el usuario antes de escribir docs
